# Wishlink

**Wishlink** — учебный Django-проект, разработанный в рамках учебной ознакомительной практики КГУ.

Проект представляет собой веб-приложение для создания и управления списками желаний. Пользователь может зарегистрироваться, войти в аккаунт, создать несколько вишлистов, добавить предметы, загрузить изображения и настроить базовую приватность списков.

## Цель проекта

Целью проекта является изучение основных возможностей Django на практике:

- создание Django-проекта и приложений;
- работа с моделями и миграциями;
- настройка авторизации пользователей;
- создание HTML-шаблонов;
- подключение CSS-стилей;
- работа со статическими и медиафайлами;
- настройка PostgreSQL;
- запуск проекта через Docker и Docker Compose;
- использование Git для контроля версий.

## Приложения проекта

### users

Приложение `users` отвечает за пользователей и профили.

Основные модели:

- `User` — кастомная модель пользователя на основе `AbstractUser`;
- `Profile` — профиль пользователя с настройками, например темой интерфейса.

### social

Приложение `social` отвечает за социальные связи между пользователями.

Основные модели:

- `FriendRequest` — заявка в друзья;
- `Relationship` — направленное отношение между пользователями.

Отношения сделаны направленными. Это значит, что отношение пользователя A к пользователю B не обязательно совпадает с отношением пользователя B к пользователю A.

### wishlists

Приложение `wishlists` отвечает за списки желаний, предметы и доступ.

Основные модели:

- `Wishlist` — список желаний;
- `WishlistItem` — предмет вишлиста;
- `WishlistAccess` — ручной доступ пользователя к конкретному вишлисту.

## Логика доступа к вишлистам

Пользователь может видеть вишлист, если выполняется одно из условий:

1. Пользователь является владельцем вишлиста.
2. Вишлист публичный.
3. Пользователю выдан ручной доступ.
4. Тип отношения пользователя входит в список разрешённых типов отношений.

Основная логика проверки доступа находится в файле:

```text
wishlists/services.py
```

## Запуск проекта через Docker

Для запуска проекта через Docker необходимо создать файл `.env` на основе `.env.example`.

Пример `.env`:

```env
SECRET_KEY=dev-secret-key
DEBUG=True

POSTGRES_DB=wishlink
POSTGRES_USER=wishlink_user
POSTGRES_PASSWORD=wishlink_password
POSTGRES_HOST=db
POSTGRES_PORT=5432
```

Запуск контейнеров:

```bash
docker compose up -d --build
```

Применение миграций:

```bash
docker compose exec web python manage.py migrate
```

Создание суперпользователя:

```bash
docker compose exec web python manage.py createsuperuser
```

После запуска сайт будет доступен по адресу:

```text
http://127.0.0.1:8000/
```

Административная панель:

```text
http://127.0.0.1:8000/admin/
```

## Локальный запуск без Docker

Также проект можно запустить локально через виртуальное окружение Python.

Создание виртуального окружения:

```bash
python -m venv .venv
```

Активация виртуального окружения на Linux:

```bash
source .venv/bin/activate
```

Установка зависимостей:

```bash
pip install -r requirements.txt
```

Применение миграций:

```bash
python manage.py migrate
```

Создание суперпользователя:

```bash
python manage.py createsuperuser
```

Запуск сервера разработки:

```bash
python manage.py runserver
```
